Now define the standard component specification template.

The template must include:

- Component name
- Purpose
- UX intent
- Visual intent
- When to use
- When NOT to use
- Anatomy (structure)
- Required parts
- Optional parts
- Variants
- Sizes
- States
- Interaction behavior
- Accessibility requirements
- Responsive behavior
- Theming behavior
- Dependencies (tokens/foundations)
- Edge cases
- Anti-patterns
- Acceptance criteria

Then apply this template in FULL DETAIL to the following components:

- Button
- Input
- Card
- Badge
- Toggle (Switch)
- Modal

IMPORTANT:
- Be extremely detailed
- Do not skip sections
- Do not be generic
- This should be implementation-ready documentation
